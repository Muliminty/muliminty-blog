## 使用 `projects` 集合

`projects` 集合位于 `src/content/projects` 目录下。

```
📁 /src/content/projects
└── 📁 project-1
      └── 📄 index.md
└── 📁 project-2
      └── 📄 index.mdx
```

在上面的示例中，将会生成两个项目文章，文件夹名称代表文章的 slug。

- `https://example.com/projects/project-1`
- `https://example.com/projects/project-2`

---

## 提供元数据

```astro
---
title: "Astro Micro";
description: "Astro Micro 是一个适用于 Astro 的无障碍主题。";
date: "2024-03-20";
draft: false;
---
```

| 字段         | 必填 | 类型     | 备注                                             |
| :----------- | :--- | :------- | :----------------------------------------------- |
| title        | 是   | string   | 文章的标题。用于 SEO 和 RSS。                    |
| description  | 是   | string   | 文章的描述。用于 SEO 和 RSS。                    |
| date         | 是   | string   | 必须是有效的日期字符串（能够被解析）。           |
| draft        | 否   | boolean  | 如果 draft: true，内容将不会被发布。             |
| demoURL      | 否   | string   | 项目演示的链接（如果有）。                       |
| repoURL      | 否   | string   | 项目仓库的链接（如果有）。                       |

---

剩下的就是在元数据下方编写内容。

```astro
---
title: "Astro Micro";
description: "Astro Micro 是一个适用于 Astro 的无障碍主题。";
date: "2024-03-20";
draft: false;
---

## Astro Micro 🔬
<!-- 内容 -->
```
